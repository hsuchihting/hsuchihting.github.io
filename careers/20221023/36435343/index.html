進擊的轉職-平面設計邁向前端工程師之奇幻旅程 第 12 話

發表於 2022-10-23 分類於 careers Disqus：
參加首屆鐵人賽，透過壓力逼迫自己成長

建立部落格

經過鐵人賽的洗禮，讓我找到一個可以記錄學習歷程的地方，原本想透過 iT 邦來當作筆記平台，透過 Ray 助教的建議，建立自己的部落格可以練習到多前端該做的事情，我使用 HEXO 來當作部落格的模板，並透過 GitHub 的版本控制操作建立部落格空間，使用指令來發布以及更新部落格文章內容，也使用工程師最喜歡的 Markdown 來撰寫文章。圖片也用 firebase 來儲存圖片，讓資源可以自己控制與掌握。

還有串接 Google Analytics 看到有多少人點擊我的部落格，意外的發現自己文章的觸角除了台灣是大宗外，也有海外的工程師參考我的撰寫的筆記文章，讓我又驚又喜。

透過部落格可以回顧每一年所走過的路，而當中的點滴只有自己知道，會滿感動的，發現自己也走了好長的路。

建立部落格的好處

累積屬於自己的文章內容，不會有其他的內容充斥其中，並且有專屬的頁面，建立個人品牌形象，文章內容比較彈性與自由，不會受到拘束，透過網址就能夠把想要分享的內容提交到平台或是朋友。有部落格後，這邊會變成前端領域的對接窗口，也讓我沒有想過我可以在資訊產業開始有一點點的人脈與人氣。此概念不只用於前端，可以延伸到其他領域中。

鼓勵寫部落格的原因

前端技術更迭速度快，範圍廣又雜，只靠腦袋記憶很容易忘記，而透過部落格的紀錄，可以直接減少去記憶的負擔，且可以把只有自己遇到的情境透過文字與程式碼記錄下來，當下次遇到相同的問題時，減少 Google 的時間，在工作上可以產生較高的效益，如果當同事有遇到相同的狀況，可以直接分享記錄過的內容，幫助他人解決曾經遇到的問題，而且透過 HEXO 部落格的程式碼架構，也可以多學習專案的建構方式，畢竟這是上線的東西，架構需要穩定，雖然撰寫部落格會花費比較多的時間，畢竟寫東西就是要把思緒釐清，並且用字遣詞也需要思考，但把部落格當作紀錄與一點出口發洩，也是很不錯的紓壓方式，且這些都是屬於自己累積的財寶，在諸多因素考量下，非常建議建立自己的部落格喔！

透過部落格把思維邏輯寫下來，盡量用第三人稱都看得懂的敘述，當過了一個月或半年回頭再來看這篇筆記，解決自己的問題，或是若一個陌生的工程師看到部落格內容獲得解惑，那這一篇就屬於成功的筆記文章。

如果想學習用 HEXO 建立部落格，非常推薦這系列文章，手把手教會如何從零建立部落格！
