進擊的轉職-平面設計邁向前端工程師之奇幻旅程 第 18 話

發表於 2022-10-23 分類於 careers Disqus：
好好的工作幹嘛離職？預期與事實有落差時的抉擇

好不容易拿到前端工作 offer，沙發還沒坐熱，怎麼急著提離職？
其實做這個決定也是掙扎很久，對於有家庭有小孩的中年男子來說，每次轉換工作都是滿大的風險，畢竟已經不是一人飽全家飽的景況。離職也有找校長以及身邊的工程師朋友給點建議，當然最後的決定還是在自己。

離職的契機

第一份工作的產業是車聯網公司，負責的專案是海外客戶，開發的產品類似打開手機或網頁會在地圖上找到車輛的系統，剛入職的時候剛好遇到此系統需要大量切版的工作，並且每兩週都要跟總經理開會會報進度，整體步調算舒服，並且不需要加班，上班時間滿八小時就可以下班，沒有主管沒走就不敢走的壓力。

但工作實際開發是用 .NET CORE 開發，沒有說這個框架不好，而是大部分主力會需要在後端開發，並且尚未發展前後端分離，也沒有使用版控工具，版本控制都是 ZIP + EMAIL…

對！你沒看錯，都已經 2020 了還這樣，加上業務面有點狀況(這不是主要考量的原因)，而是在開發層面只有我一人前端，對於個人技術成長似乎有限，雖然前輩們都建議我待個半年一年累積點經驗，但我自己評估這邊之後會往後端發展，加上年紀關係沒有大把時間這樣慢慢磨，故最後禱告找尋內心的答案後提出辭呈。

那要去哪？

當我入職兩個多月時，主管肯定我的表現與產出，提早讓我通過試用期，這對我來說又是一劑強心針，但礙於技術成長的考量，我還是試著投遞履歷看看有無新的機會，沒想到在先前一間很想去的系統商主動聯繫是否報到？

畢竟是當時心之所向的公司，提到使用的框架是 Angular，並且不是救火員，再三確認如果是救火員我沒有此意願報到，畢竟我不希望跳槽是一種入火坑的概念，結局是直接當砲灰，聽完 package 後與工作內容後考慮了幾天，最後答應前往報到，提出離職後，主管安排我完成最後幾項需要切版的工作，後面幾天就讓我自主進修，我便學習下份工作所需要的知識，並帶著一知半解的框架知識前往下一站。

沒有最好的決定，是決定後做到最好

這間公司樓下的美食街有一間咖啡，某天店外的板子上寫著這段話，給我很大的啟示，許多的決定都是如此，總是不夠完美。

但這世上沒有最完美的決定，也沒有最好的決定，而是不要做出後悔的決定，以及決定後就要勇敢的面對下一趟的旅程，把接下來面對的事情做到最好。
