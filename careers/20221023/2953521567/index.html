進擊的轉職-平面設計邁向前端工程師之奇幻旅程 第 15 話

發表於 2022-10-23 分類於 careers Disqus：
面對年齡的思考與能力上要評估的思維

當我決定要轉職前端工程師時已經是 36 歲了，二月總共面試了八間公司，九月被資遣後面試了六間公司，十二月二度資遣後面試三間，畢竟中年轉職除了內心會有點自卑還有莫名的自尊外，外在現實仍需要面對，我整理了一些面試心得，用條列式呈現，給跟我同樣年紀的朋友參考，從原本熟悉的領域要跨到全新資訊領域時，健康的心理建設才是更重要的。

面試心得記錄

  1. 不要被年紀限制，雖然它很現實。
  2. 作品可以幫自己說話，好好打磨面試作品，並且要很有自信的講解。
  3. 開缺說是前端工程師，但很多跟前端工程師的內容有差距，必要時還是會需要碰到後端，務必要了解清楚。
  4. JS 務必熟練，建議把核心篇買起來，未來框架很吃重陣列物件的操作觀念。
  5. JS 不熟練之前，別急著碰 JS 框架，不然兩頭空。
  6. 面試的時候，公司選你，你也在選公司，不要妄自菲薄，謙卑但不自卑。
  7. 工程師要有自學的能力，最好都要能獨立作業，這對沒有開發經驗的我們來說，是最大的挑戰。
  8. 必問：「前面一份工作看你做得滿久的，為什麼要轉職?」畢竟到這年紀已經不是新鮮人，轉職必需要有著力點的理由！
  9. 每個產業都需要前端工程師，建議在面試前先行上網了解公司背景與內容，對了! 對方公司也會調查你。
  10. 最好搭上主流 ABC 的領域，會比較有發揮與成長空間。A: AI, B: Big Data, C: Cloud
  11. 面試完得失心不要太重，以免挫折感太強，沒被錄取並非被否定。
  12. 整理好自己該進步的地方，再跟著教學影片跟練習做一次，一定要寫成筆記。
  13. 養成有問題卡關不要太久，建議不要超過 30 分鐘，因為目前遇到的機上考試都只有 30 分鐘，若進入職場開發專案也會有時程上的壓力(這並非絕對時間，要看公司可以給予的空間)。
  14. 不要鑽程式的牛角尖，同個思維久了會想法閉塞，做點別的事情之後再回來看就會懂了。
  15. 養成寫部落格的習慣，對自己的幫助是最大的。
  16. 記得寫一份 cakeresume 的履歷，會很加分！
  17. 會就會，不會就不會，誠實就好。
  18. 對自己的作品要有自信。
  19. 要喜歡現在所做的一切，才不會被環境打敗!
  20. 薪水固然重要，但第一份或是要跳槽，要各方面評估，不要只考慮薪水，成長性更重要!
  21. 面試服裝乾淨整齊，前一天要確認通勤時間，並要提早 10 分鐘抵達面試地點。
  22. 面試回話，慢慢回，等對方說完再答話，不要搶話，就算面試官可能理解錯誤或是偏離話題，都等他講完再回話。
  23. 如果可以的話，JavaScript 演算法的相關觀念可以慢慢補足。
  24. JSON / AJAX 的觀念與相關使用經驗。
  25. Git 版本控制會問到有沒有使用經驗。
  26. 協同開發會使用 GitLab 或 BitBucket(看公司)做版本控制。
  27. 全域變數與區域變數在函式的觀念。
  28. 過去工作經驗中是否有比較困難的經驗? 如何解決? 檢討後的執行方式是什麼?
  29. 過去工作經驗中是否有比較成功的案例? 實質幫助公司什麼地方?
  30. 遇到工作上有衝突時怎麼處理?
  31. 每一間公司都會需要三到五分鐘的自我介紹。(這時候在準備 cakeresume 履歷就很重要啊!)
  32. 因為瀏覽器支援關係，請寫出垂直置中至少三種寫法，如果不用 flex 的方式也依定要會，float 沒有被棄用，一定要會。
  33. 不同公司會因為專案使用不同的框架，但建議還是掌握一個主流框架起來，觀念基本上會差不多，學習別的框架也會比較快進入狀況，因為進公司可能專案需求會需要學新的框架。
  34. 如果有需要先行填寫資本資料，身分證跟地址不一定要寫，但如果是進公司寫的，建議還是填寫一下，但地址不用寫得很完整，因為公司只是要知道你大概住哪裡跟交通位置而已。
  35. HTML, CSS, JavaScript 基礎穩定最為重要。
  36. 寫程式已經很苦悶了，面談時請表現出積極且熱情的樣子，有加分。
  • 雖然疫情已經趨緩，但當時有提到如果口罩地圖做得出來算是很不錯的工程師!!
  • 後台介面是一定會碰到的專案，table 與 form 的觀念的實作可以練習看看，會有這個需求！
  • 鼓勵大家寫好一份 cakeresume 履歷分別整理好放到你習慣的人力平台上，除了主動出擊外，也增加被看到的機會。
  • cover letter 也請一定要依照公司客製化，不要使用預設內容就投遞所有的公司。
